# AgentRunner — Coding Agent User Journeys 目录

以 Claude Code（终端/交互）与 Codex cloud（异步/云端）为标尺的 20 条
user journey，从基础到复杂，覆盖 coding agent 的完整能力面。每条 10 秒
可读：一句场景流程 + 一句考验点。这是**目录**（做什么）；对照 DESIGN
的缺口分析记在 GAPS.md（缺什么）。

---

## A · 日常交互（终端，Claude Code 形态）

### UJ-01 即问即答 `基础`
"这个 repo 的鉴权在哪做的？" → agent 搜码读文件 → 引用出处作答，零改动。
**考验**：只读工具面、检索质量、不碰 workspace。

### UJ-02 小修快跑 `基础`
"这个测试挂了，修一下" → 读错误 → 改一处 → 跑测试变绿 → 汇报。
**考验**：编辑+执行闭环、失败重试、结果如实汇报。

### UJ-03 结对续聊 `基础`
答完一轮，用户追问"为什么这么改？"→ 同一上下文继续 → 再追问再改，聊一下午。
**考验**：会话续聊形态（answer 后等下一条消息）、上下文连续性。

### UJ-04 贴图贴日志 `基础`
用户贴一张报错截图 + 500 行日志 → agent 看图读日志定位 → 修复。
**考验**：图片/附件输入、长粘贴内容的处理。

### UJ-05 从零起项目 `基础`
"起一个 Go 的 CLI 项目，带 CI" → 脚手架、依赖、首个测试绿、README。
**考验**：空目录 workspace、多文件创建、约定选型。

### UJ-06 大重构走计划 `进阶`
"把回调改成 async" → plan mode 出方案 → 用户批准 → 分步执行，每步测试护航。
**考验**：plan mode、计划审批载荷、长执行的阶段推进。

### UJ-07 中途纠偏 `进阶`
任务跑到一半，用户："等等，用 v2 的 API"→ agent 收到即调头；再一条"顺便改日志级别"排队生效。
**考验**：steering 消息、type-ahead 队列、interrupt 与 steer 的分工。

### UJ-08 权限日常 `进阶`
agent 要跑 `rm -rf build` → 弹审批 → 用户"允许且不再问" → 后续同类直过；`curl` 被网络沙箱拦下。
**考验**：审批流、规则运行时持久化、沙箱红线。

### UJ-09 长会话续命 `进阶`
聊了 300 turn 上下文快满 → 自动/手动 compaction → "记住这项目用 pnpm"入项目记忆 → 明天 resume 接着来。
**考验**：compaction、记忆写回、跨日 resume。

## B · Git 与协作

### UJ-10 提交流水 `基础`
改完代码 → agent 起分支、写 commit message、push、开 PR 带描述。
**考验**：git 工作流、PR 创建、提交纪律（不碰 main 类约束）。

### UJ-11 代码评审员 `进阶`
"审一下 PR #42" → agent 拉 diff、逐文件找问题 → 输出 findings 列表，不改一行码。
**考验**：只评不改的角色约束、diff 理解、结构化输出。

### UJ-12 PR 保姆 `高级`
"盯着这个 PR 到能合" → CI 红灯事件唤醒 → 诊断推修复 → reviewer 评论逐条处理 → 用户插话"第 3 条不用理" → 绿灯后通知。
**考验**：外部事件唤醒既有 session、事件驱动循环、人机混合插话。

## C · 异步与云端（Codex cloud 形态）

### UJ-13 手机派活 `高级`
手机上对 repo 提任务 → 云容器 provision（clone+setup+网络策略）→ agent 跑完出 diff → 网页审阅 → 开 PR → 次日"改成 t.Parallel"同环境续作。
**考验**：环境生命周期、远程提交/审阅、diff→PR 链路、follow-up 续聊。

### UJ-14 定时值守 `进阶`
"每晚 2 点跑依赖审计，有 CVE 就修并开 PR" → cron 驱动 → 无事静默，有事干活并通知。
**考验**：cron/loop 驱动、overlap 策略、静默与通知的分寸。

### UJ-15 通宵冲目标 `高级`
睡前"覆盖率提到 80%"→ goal 驱动 + verifier 打分 + 预算上限 → 夜里某轮改崩分数下跌 → 晨起看时间线，rewind 到坏轮之前换思路。
**考验**：goal/verifier/停滞检测、预算、时间线复盘、rewind/fork。

### UJ-16 三路并击 `高级`
难题一次派三个隔离尝试（best-of-3）→ verifier 选优 → 胜者 diff 晋升回主线。
**考验**：并行隔离 worktree、判分选优、胜者晋升语义。

### UJ-17 远程驾驶舱 `进阶`
云端任务跑着，用户在网页看直播 → 发现走偏发一条 steer → 没救了直接 stop → 用量与判定链全程可查。
**考验**：attach 直播、远程 steer/stop 控制面、审计可见性。

## D · 规模化与生态

### UJ-18 多 agent 编排 `高级`
bug 报告+截图 → 主 agent 并行派 3 个子 agent（复现/查历史/查依赖），自己继续读码 → 用户 steer"别查依赖了，查迁移文档" → 杀一个、新起一个 → 全部回灌后汇总。
**考验**：后台并行子 agent、handle/kill、steer 重定向、收敛汇总。

### UJ-19 生态接入 `进阶`
"按 JIRA-123 的需求实现" → MCP 连 ticket 系统读需求 → 按 repo 的 skills/自定义命令干活 → 完成后回写 ticket 评论。
**考验**：MCP 生命周期与工具面、skills 注入、外部系统读写审批。

### UJ-20 不受信审计 `高级`
clone 陌生 repo"帮我看这库能不能用" → 首次信任决策 → README 里埋着注入指令诱导外传 .env → 沙箱与凭据红线兜住 → 事后审计它动过什么。
**考验**：信任模型、注入对抗、网络/凭据红线、全程审计。
