# 协调者本人:web 驾驶舱黑盒 pass(浏览器实测,新用户视角)

前提校准:web app 自称"arweb 测试驾驶舱"(README),是给开发者驱动 AgentRunner 会话的**测试工具**,不是消费级产品。所以"原始 journal 显示原始 JSON""inspect 树是内部结构"这类是 by-design 的调试面,不当 basic-workflow bug 记。但功能性 bug 与消费级产品无关,照记。

## 正面(web 相比裸 CLI 大幅更好)
- 新会话弹窗**预填能用的 spec**(带 tools + permissions allow)+ "造空 workspace" 按钮 → 绕过 CLI 的 spec-无从发现 / 工具默认禁两堵墙。
- **内联显示 AI 回复 + 工具调用卡片**(可展开参数/结果)→ 解决 CLI 的"回复不显示"。
- 有可见的 **图片** 按钮(挨着 发送)→ 传图在 web 里可发现(对比 CLI:传图几乎无从发现,见 R2-C-2)。
- C1 修复在 web 上验证通过(见上一轮 web 端到端测试:杀死会话的原始序列现走完全程)。

## 功能性问题(fresh /tmp/ar C1-fixed 代码,8801 实例,浏览器实测)
### [R2-W-1] "关闭会话"按钮点了完全无反应,零反馈 🟠
点右上 关闭会话 → 等 4 秒,header 与侧栏状态仍 `待命,等你输入`、13 轮不变;无确认、无 toast、无报错、无 console 报错。用户**无法从驾驶舱关闭一个会话**,也不知道点了有没有用。与 CLI 黑盒"close 关不掉"同源(daemon 持有的 live 会话),但 web 连"closing"回执都不给。

### [R2-W-2] 新会话点"创建"后弹窗不关闭,会话静默建在弹窗背后 🟠
点 创建 → 等 6 秒,侧栏出现新会话(1 轮,已真跑),但 新会话 弹窗**仍开着**、workspace 字段还在。用户看不到会话已建(被弹窗挡着),易①以为没成功而**再点创建→建重复会话**,②困惑。期待:创建后弹窗关闭并跳到新会话。

### [R2-W-3](观察,非 bug)daemon 会话存储跨 web 实例共享
我在 8801 全新 runtime 起的实例,侧栏却出现我之前在 8799 建的会话——web 的 runtime 目录只管便利产物,会话数据在 daemon 的 XDG(默认、共享)。对"测试驾驶舱"无碍,记录以备。
